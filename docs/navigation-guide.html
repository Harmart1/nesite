1. Overview of Navigation Components

The website utilizes several navigation components working together:

  • Main Navigation: The primary site navigation in the header.
  • Jump Navigation: On-page navigation for practice area pages.
  • Breadcrumb Navigation: For clear hierarchy display and backward navigation.
  • Continuous Scroll Navigation: Including progress bar and "Continue Reading" buttons.
  • Footer Navigation: Supplementary navigation links organized by category.

2. Main Navigation Setup

The main navigation is implemented in the site header across all pages:

HTML Structure

<nav class="main-nav">
    <button class="mobile-menu-toggle" aria-label="Toggle Navigation Menu">
        <span></span><span></span><span></span>
    </button>
    <ul class="nav-menu">
        <li><a href="index.html">Home</a></li>
        <li><a href="about.html">About</a></li>
        <li class="dropdown">
            <a href="practice-areas.html">Practice Areas</a>
            <ul class="dropdown-menu">
                <li><a href="civil-litigation.html">Civil Litigation</a></li>
                <li><a href="business-law.html">Business Law</a></li>
                <li><a href="technology-law.html">Technology Law</a></li>
                <li><a href="general-practice.html">General Practice</a></li>
            </ul>
        </li>
        <li><a href="resources.html">Resources</a></li>
        <li><a href="blog/index.html">Blog</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

Adding New Items

To add a new top-level navigation item:

  1. Create the new page in the root directory
  2. Add a new <li> element to the .nav-menu list
  3. For dropdown menus, follow the pattern of the "Practice Areas" item

Note: The nav-highlight.js script will automatically add the active class to the current page's navigation item.

3. On-Page Navigation Setup

For content-heavy pages like practice areas, on-page navigation is implemented via jump links:

Jump Navigation HTML

<div class="jump-nav">
    <div class="container">
        <ul class="jump-nav-links">
            <li><a href="#overview" class="active">Overview</a></li>
            <li><a href="#services">Services</a></li>
            <li><a href="#approach">Our Approach</a></li>
            <li><a href="#industries">Industries</a></li>
            <li><a href="#faq">FAQ</a></li>
        </ul>
    </div>
</div>

Section IDs

Each section referenced in the jump navigation must have a matching ID:

<section id="overview" class="page-section">
    <!-- Section content -->
</section>

Important: Always ensure that section IDs exactly match the href attribute values in your jump navigation links (minus the # prefix).

4. Breadcrumb Implementation

Breadcrumbs should be added to all inner pages to provide hierarchical context:

How to Add Breadcrumbs

  1. Copy the breadcrumb component from /components/breadcrumbs.html
  2. Paste it immediately after the header in your page
  3. Update the path and title values to reflect the page's position in the site hierarchy

Example implementation:

<nav aria-label="Breadcrumb" class="breadcrumbs">
    <div class="container">
        <ol class="breadcrumb-list">
            <li><a href="index.html">Home</a></li>
            <li><a href="practice-areas.html">Practice Areas</a></li>
            <li aria-current="page">Business Law</li>
        </ol>
    </div>
</nav>

CSS Requirements

Ensure the breadcrumbs.css file is included in the page's head:

<link rel="stylesheet" href="css/breadcrumbs.css">

5. Continuous Scroll Navigation

Practice area pages use continuous scroll navigation with progress tracking:

Progress Bar

Add this code immediately after the opening <body> tag:

<!-- Page progress indicator -->
<div class="page-progress">
    <div class="page-progress-bar" id="progressBar"></div>
</div>

Continue Reading Buttons

Add these buttons at the end of each section to encourage scrolling to the next section:

<div class="continue-reading">
    <button class="continue-button" aria-label="Continue to Next Section" data-target="#next-section-id">
        <span>Continue to Next Section</span>
        <svg class="arrow" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="currentColor">
            <path d="M7.41 8.59L12 13.17l4.59-4.58L18 10l-6 6-6-6 1.41-1.41z"/>
        </svg>
    </button>
</div>

Note: Replace #next-section-id with the actual ID of the next section.

Required JavaScript

Ensure these scripts are included before the closing </body> tag:

<script src="js/scroll-navigation.js" defer></script>
<script src="js/nav-highlight.js" defer></script>

6. Mobile Navigation Considerations

Mobile navigation has special requirements:

Mobile Menu Toggle

The mobile menu toggle button is already part of the main navigation structure. Its functionality is provided by mobile-navigation.js.

Horizontal Scrolling Jump Navigation

On mobile devices, the jump navigation becomes horizontally scrollable:

  • No additional code is needed as this is handled by CSS in responsive.css
  • The nav-highlight.js script will automatically scroll the active link into view

Touch-Friendly Navigation

Mobile-specific touch handling is implemented in mobile-navigation.js. This script enables:

  • Touch-based scrolling for horizontal jump navigation
  • Improved tap targets for dropdown navigation
  • Optimized scroll offsets for mobile viewport heights

Tip: Always test navigation changes on mobile devices to ensure touch interactions work properly.
